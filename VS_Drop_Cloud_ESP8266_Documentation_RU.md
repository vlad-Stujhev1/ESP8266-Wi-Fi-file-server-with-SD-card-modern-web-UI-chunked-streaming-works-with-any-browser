 * │   │ (Смартфон)  │◄────►│   Server    │◄────►│  (Файлы)    │            │
 * │   └─────────────┘ WiFi └─────────────┘ SPI  └─────────────┘            │
 * │         │                     │                                        │

 * │         ▼                     ▼                                        │
 * │   ┌─────────────┐      ┌─────────────┐                                 │

 * 
 * ПОСЛЕДОВАТЕЛЬНОСТЬ ЗАПРОСА:
 *

 * 2. Клиент открывает браузер и вводит http://192.168.4.1
 * 3. ESP8266 принимает HTTP запрос через ESP8266WebServer
 * 4. WebServer парсит URL и вызывает соответствующий handler
 * 5. Handler читает/пишет файлы на SD-карту через SPI
 * 6. Handler формирует HTML ответ (из Flash памяти)
 * 7. WebServer отправляет ответ клиенту

 *      │                              │                          ▲
 *      │                              │                          │
 *      │  1. Выбор файла              │  2. Приём чанков         │
 *      │  2. Отправка формы           │  3. Запись на SD         │
 *      │  3. Прогресс-бар             │  4. Подтверждение        │
 *      ▼                              ▼                          │
 *                                                                   │
 *                                                                 3. Потоковая запись
 * 
 * СКАЧИВАНИЕ ФАЙЛА (Download):
 * 
 * ┌──────────┐                   ┌──────────┐    read()    ┌──────────┐
 * │  Клиент  │ ◄─────────────── │  ESP8266 │ ◄─────────── │ SD-карта │
 * │          │   HTTP Response   │ streamFile│   chunks    │          │
 * │  Файл    │   MIME type       │           │             │  Файл    │
 * └──────────┘                   └──────────┘              └──────────┘
 *      │                               │                         │
 *      │  1. GET /download?path=file   │                         │
 *      │  2. Приём файла               │  3. Чтение чанков       │

 * 
 * 
 * ВАЖНО: ESP8266 НЕ загружает весь файл в память!
 *        Используется streaming - чтение/запись по 8KB chunks.
 *        Это позволяет работать с файлами ЛЮБОГО размера.
 *

/*
 * 2.1. ХАРАКТЕРИСТИКИ ЧИПА
 * -------------------------
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │                    ESP8266EX Specifications                     │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  CPU:         Tensilica L106 32-bit RISC                        │
 * │               80 MHz (по умолчанию)                             │
 * │               160 MHz (разгон, нестабильно)                     │
 * │                                                                 │
 * │  RAM:         160 KB total                                      │
 * │               ├── 32 KB для инструкций (IRAM)                   │
 * │               ├── 32 KB для данных (DRAM)                       │
 * │               └── 96 KB для WiFi стека (не доступна!)           │
 * │                                                                 │
 * │               РЕАЛЬНО ДОСТУПНО: ~40-50 KB для вашей программы!  │
 * │                                                                 │
 * │  Flash:       Внешняя, 1-16 MB                                  │
 * │               Код выполняется через кэш (XIP)                   │
 * │               Данные читаются напрямую                          │
 * │                                                                 │
 * │  WiFi:        802.11 b/g/n                                      │
 * │               2.4 GHz only                                      │
 * │               До 72 Mbps PHY rate                               │
 * │               TCP/IP стек: lwIP                                 │
 * │                                                                 │
 * │  SPI:         4 слейва максимум                                 │
 * │               До 40 MHz clock                                   │
 * │               Используется для Flash и SD карты                 │
 * │                                                                 │
 * │  GPIO:        17 пинов                                          │
 * │               6 для Flash (заняты!)                             │
 * │               11 доступно (с ограничениями)                     │
 * │                                                                 │
 * │  ADC:         10-bit, 1 канал (0-1.0V!)                         │
 * │                                                                 │
 * │  UART:        2 порта (UART0 для Serial, UART1 для debug)       │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * 
 * MEMORY MAP ESP8266:
 * 
 * ┌────────────────────┬──────────────┬─────────────────────────────┐
 * │      Region        │    Size      │         Description         │
 * ├────────────────────┼──────────────┼─────────────────────────────┤
 * │ 0x40100000-...     │ 32 KB        │ IRAM (Instructions RAM)     │
 * │ 0x40200000-...     │ 1 MB mapped  │ Flash (XIP - execute in place)│
 * │ 0x3FFE8000-...     │ 32 KB        │ DRAM (Data RAM)             │
 * │ 0x40110000-...     │ 96 KB        │ WiFi Buffer (использует SDK)│
 * └────────────────────┴──────────────┴─────────────────────────────┘
 * 
 */


/*
 * 2.2. ОГРАНИЧЕНИЯ ПАМЯТИ - КРИТИЧЕСКИ ВАЖНО!
 * -------------------------------------------
 * 
 * ПОЧЕМУ ЭТО ВАЖНО:
 * 
 * ESP8266 имеет ОЧЕНЬ мало RAM. При неаккуратном программировании
 * программа вылетит с ошибкой "Exception 0" или просто перезагрузится.
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │              ТИПИЧНОЕ ИСПОЛЬЗОВАНИЕ RAM                         │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  WiFi Stack:              ~24 KB (всегда занято)                │
 * │  lwIP TCP/IP:             ~8 KB                                │
 * │  System:                  ~4 KB                                │
 * │  HTTP Server buffers:     ~4 KB                                │
 * │  ─────────────────────────────────                             │
 * │  ИТОГО занято системой:   ~40 KB                               │
 * │                                                                 │
 * │  СВОБОДНО ДЛЯ ВАШЕГО КОДА: ~40 KB                               │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * 
 * ГЛАВНЫЕ ВРАГИ RAM:
 * 
 * 1. String класс - динамическое выделение памяти
 *    ПЛОХО:
 *    String bigString = "Очень длинная строка...";
 *    
 *    ХОРОШО:
 *    Serial.println(F("Очень длинная строка...")); // F() хранит во Flash
 * 
 * 2. Глобальные переменные - занимают RAM постоянно
 *    ПЛОХО:
 *    char buffer[8192];  // 8 KB занято навсегда!
 *    
 *    ХОРОШО:
 *    Использовать стек или не использовать большие буферы вообще
 * 
 * 3. Конкатенация строк - фрагментация памяти
 *    ПЛОХО:
 *    String html = "<html>";
 *    html += "<head>";
 *    html += "<title>";
 *    // Каждый += выделяет новую память!
 *    
 *    ХОРОШО:
 *    Хранить HTML во Flash (PROGMEM)
 * 
 * 
 * РЕШЕНИЕ: PROGMEM И F() МАКРОС
 * 
 * PROGMEM - специальный атрибут, указывающий компилятору разместить
 * данные во Flash-памяти, а не в RAM.
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │                   Flash vs RAM                                  │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  RAM:    50 KB    - Быстро, но мало                            │
 * │           ▲         Для переменных и стека                      │
 * │           │                                                     │
 * │  Flash:  4 MB     - Медленнее, но много                        │
 * │           ▲         Для кода и констант                         │
 * │           │                                                     │
 * │  PROGMEM хранит данные во Flash, освобождая RAM                │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ:
 * 
 * // 1. F() макрос - для строк в Serial.print
 * Serial.println(F("Эта строка во Flash")); // 0 байт RAM
 * Serial.println("Эта строка в RAM");       // ~30 байт RAM
 * 
 * // 2. PROGMEM - для больших массивов и строк
 * const char CSS_STYLES[] PROGMEM = R"=====(
 *   body { background: #000; }
 *   ... тысячи символов CSS ...
 * )=====";
 * 
 * // 3. FPSTR() - для чтения PROGMEM строк
 * html += FPSTR(CSS_STYLES);  // Читает из Flash
 * 
 * // 4. PSTR() - для inline строк
 * Serial.println(PSTR("Inline Flash string"));
 * 
 */


/*
 * 2.3. ПЕРИФЕРИЯ: SPI И WIFI
 * ---------------------------
 * 
 * SPI (Serial Peripheral Interface):
 * ──────────────────────────────────
 * 
 * Используется для общения с SD-картой.
 * 
 * ┌───────────────────────────────────────────────────────────────┐
 * │                  SPI Pins на ESP8266                          │
 * ├───────────────────────────────────────────────────────────────┤
 * │                                                               │
 * │  Signal   │  GPIO  │  Pin   │  SD Card   │  Direction        │
 * │───────────┼────────┼────────┼────────────┼───────────────────│
 * │  MOSI     │  13    │  D7    │  MOSI      │  ESP → SD         │
 * │  MISO     │  12    │  D6    │  MISO      │  SD → ESP         │
 * │  SCK      │  14    │  D5    │  SCK       │  ESP → SD         │
 * │  CS       │  4     │  D2    │  CS        │  ESP → SD         │
 * │                                                               │
 * │  MOSI = Master Out Slave In (данные от мастера к слейву)     │
 * │  MISO = Master In Slave Out (данные от слейва к мастеру)     │
 * │  SCK  = Serial Clock (тактовый сигнал)                       │
 * │  CS   = Chip Select (активация устройства)                   │
 * │                                                               │
 * └───────────────────────────────────────────────────────────────┘
 * 
 * ВАЖНО: SD карта работает на 3.3V!
 *        Подключение к 5V уничтожит карту!
 * 
 * 
 * СКОРОСТЬ SPI:
 * 
 * SD.begin(CS, SPI_HALF_SPEED);    // ~6 MHz - стабильно
 * SD.begin(CS, SPI_QUARTER_SPEED); // ~3 MHz - для плохих карт
 * SD.begin(CS, SPI_EIGHTH_SPEED);  // ~1.5 MHz - для очень плохих карт
 * 
 * Почему HALF_SPEED? 
 * - На максимальной скорости SD карты часто дают ошибки
 * - Дешёвые китайские модули имеют плохую разводку
 * - HALF_SPEED - золотая середина стабильности и скорости
 * 
 * 
 * WiFi SoftAP (Точка доступа):
 * ────────────────────────────
 * 
 * В режиме AP ESP8266 создаёт свою WiFi сеть.
 * 
 * ┌───────────────────────────────────────────────────────────────┐
 * │                  SoftAP Configuration                         │
 * ├───────────────────────────────────────────────────────────────┤
 * │                                                               │
 * │  WiFi.mode(WIFI_AP);           // Режим точки доступа        │
 * │  WiFi.softAP(ssid, password);  // Создать сеть               │
 * │                                                               │
 * │  IP адрес ВСЕГДА: 192.168.4.1                                │
 * │  DHCP диапазон:    192.168.4.2 - 192.168.4.x                 │
 * │  Максимум клиентов: 4-8 (зависит от SDK)                     │
 * │                                                               │
 * └───────────────────────────────────────────────────────────────┘
 * 
 * Преимущества AP режима:
 * 
 * 1. Автономность - не нужен роутер
 * 2. Предсказуемость - всегда один IP (192.168.4.1)
 * 3. Простота - клиент просто подключается к сети
 * 4. Безопасность - сеть изолирована от интернета
 * 
 * Недостатки:
 * 
 * 1. Нет интернета для клиентов
 * 2. Ограниченная дальность
 * 3. Только 2.4 GHz
 * 
 */

 *        ├── SPI.begin()                                        │
 *        │   Инициализация SPI для SD карты                     │
 *        │                                                      │
 *        ├── SD.begin(CS_PIN)                                   │
 *        │   Монтирование файловой системы                      │
 *        │                                                      │
 *        ├── WiFi.softAP(ssid, password)                        │
 *        │   Запуск точки доступа                               │
 *        │                                                      │
 *        ├── server.on("/", handleRoot)                         │
 *        │   Регистрация обработчиков URL                       │
 *        │                                                      │
 *        └── server.begin()                                     │
 *            Запуск HTTP сервера                                │
 *                                                               │
 *                                                               ▼
 * ┌──────────────┐     ┌─────────────────────────────────────────┐
 * │    loop()    │ ──► │  while(true) {                          │
 * └──────────────┘     │    server.handleClient(); // HTTP reqs  │
 *        │             │    yield();              // Watchdog    │
 *        └─────────────│  }                                      │
 *                      └─────────────────────────────────────────┘
 * 
 */


/*
 * 3.2. ОПТИМИЗАЦИИ ПАМЯТИ
 * ------------------------
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │              Техники оптимизации в прошивке                     │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  1. PROGMEM ДЛЯ HTML/CSS                                        │
 * │     ─────────────────────────────────────────────────────────  │
 * │                                                                 │
 * │     // HTML хранится во Flash, не в RAM                        │
 * │     const char CSS_STYLES[] PROGMEM = R"=====                  │
 * │       body { background: #0a0a0a; }                            │
 * │       ... тысячи символов ...                                  │
 * │     )=====";                                                   │
 * │                                                                 │
 * │     // Чтение через FPSTR():                                   │
 * │     html += FPSTR(CSS_STYLES);  // Не копирует в RAM!          │
 * │                                                                 │
 * │     // Экономия: ~5 KB CSS не занимают RAM                     │
 * │                                                                 │
 * │                                                                 │
 * │  2. F() МАКРОС ДЛЯ СТРОК                                        │
 * │     ─────────────────────────────────────────────────────────  │
 * │                                                                 │
 * │     // Вместо:                                                 │
 * │     Serial.println("Error: SD card not found");  // В RAM     │
 * │                                                                 │
 * │     // Используем:                                             │
 * │     Serial.println(F("Error: SD card not found")); // Flash   │
 * │                                                                 │
 * │     // Экономия: ~30 байт на каждую строку                     │
 * │                                                                 │
 * │                                                                 │
 * │  3. STRING RESERVE()                                            │
 * │     ─────────────────────────────────────────────────────────  │
 * │                                                                 │
 * │     // Когда String всё же нужен:                              │
 * │     String html;                                                │
 * │     html.reserve(4096);  // Выделить память один раз           │
 * │                                                                 │
 * │     // Вместо множественных перераспределений                  │
 * │                                                                 │
 * │                                                                 │
 * │  4. ИЗБЕГАТЬ ГЛОБАЛЬНЫХ БУФЕРОВ                                 │
 * │     ─────────────────────────────────────────────────────────  │
 * │                                                                 │
 * │     // ПЛОХО:                                                  │
 * │     char globalBuffer[8192];  // 8 KB занято навсегда          │
 * │                                                                 │
 * │     // ХОРОШО:                                                 │
 * │     void processFile() {                                       │
 * │       char localBuffer[512];  // В стеке, освободится          │
 * │     }                                                           │
 * │                                                                 │
 * │                                                                 │
 * │  5. STREAMING ВМЕСТО БУФЕРИЗАЦИИ                                │
 * │     ─────────────────────────────────────────────────────────  │
 * │                                                                 │
 * │     // ПЛОХО: Загрузить весь файл в память                    │
 * │     String content = file.readString();                        │
 * │     server.send(200, "text/html", content);                    │
 * │                                                                 │
 * │     // ХОРОШО: Потоковая передача                              │
 * │     server.streamFile(file, "text/html");                      │
 * │     // Читает и отправляет чанками по ~1460 байт              │
 * │                                                                 │
 * │                                                                 │
 * │  6. SERIALPRINTF_P ДЛЯ ФОРМАТИРОВАНИЯ                           │
 * │     ─────────────────────────────────────────────────────────  │
 * │                                                                 │
 * │     // Форматные строки во Flash                               │
 * │     Serial.printf_P(PSTR("[UPLOAD] %s: %u bytes\n"),          │
 * │                      filename, size);                          │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * 
 * СРАВНЕНИЕ ПОТРЕБЛЕНИЯ ПАМЯТИ:
 * 
 * ┌──────────────────────────┬────────────────┬────────────────┐
 * │        Подход            │     RAM        │     Flash      │
 * ├──────────────────────────┼────────────────┼────────────────┤
 * │  HTML в String           │  ~15 KB        │  -             │
 * │  HTML в PROGMEM          │  ~100 B        │  ~15 KB        │
 * ├──────────────────────────┼────────────────┼────────────────┤
 * │  Serial.print("text")    │  len(text)     │  -             │
 * │  Serial.print(F("text")) │  ~0            │  len(text)     │
 * ├──────────────────────────┼────────────────┼────────────────┤
 * │  file.readString()       │  file.size     │  -             │
 * │  streamFile()            │  ~1.5 KB       │  -             │
 * └──────────────────────────┴────────────────┴────────────────┘
 * 
 */


/*
 * 3.3. WEB SERVER МЕХАНИЗМ
 * -------------------------
 * 
 * ESP8266WebServer работает на основе callback-функций.
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │                  HTTP Request Processing                        │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  1. Клиент отправляет HTTP запрос                              │
 * │     GET /?path=/documents HTTP/1.1                             │
 * │                                                                 │
 * │  2. ESP8266WebServer принимает соединение                      │
 * │     server.handleClient() в loop()                             │
 * │                                                                 │
 * │  3. Парсинг запроса                                            │
 * │     - Method: GET, POST, etc.                                  │
 * │     - URL: /?path=/documents                                   │
 * │     - Arguments: path=/documents                               │
 * │     - Headers: Cookie, Authorization                           │
 * │                                                                 │
 * │  4. Поиск обработчика (Routing)                                │
 * │     server.on("/", handleRoot);  // Зарегистрированный route   │
 * │                                                                 │
 * │  5. Вызов callback-функции                                     │
 * │     handleRoot() формирует ответ                               │
 * │                                                                 │
 * │  6. Отправка ответа                                            │
 * │     server.send(200, "text/html", html);                       │
 * │     или server.streamFile(file, mimeType);                     │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * 
 * РОУТИНГ (Routing):
 * 
 * // Простые роуты
 * server.on("/", handleRoot);                    // GET /
 * server.on("/login", HTTP_GET, handleLogin);    // GET /login
 * server.on("/login", HTTP_POST, handleLoginPost); // POST /login
 * 
 * // Роут с параметрами
 * // URL: /download?path=/file.txt
 * server.on("/download", handleDownload);
 * // В handler: server.arg("path") → "/file.txt"
 * 
 * // File upload
 * server.on("/upload", HTTP_POST, 
 *           handleUploadDone,    // Вызывается в конце
 *           handleUploadPost);   // Вызывается для каждого чанка
 * 
 * // 404 handler
 * server.onNotFound(handleNotFound);
 * 
 * 
 * МУЛЬТИПАРТ ЗАГРУЗКА ФАЙЛОВ:
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │                Multipart Upload Flow                            │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  Клиент отправляет:                                            │
 * │  POST /upload HTTP/1.1                                         │
 * │  Content-Type: multipart/form-data; boundary=----WebKitForm    │
 * │                                                                 │
 * │  ------WebKitForm                                              │
 * │  Content-Disposition: form-data; name="file"; filename="a.jpg" │
 * │  Content-Type: image/jpeg                                      │
 * │                                                                 │
 * │  <бинарные данные файла>                                       │
 * │  ------WebKitForm--                                            │
 * │                                                                 │
 * │  ESP8266WebServer парсит и вызывает:                           │
 * │                                                                 │
 * │  1. UPLOAD_FILE_START                                          │
 * │     upload.filename = "a.jpg"                                  │
 * │     upload.totalSize = 0                                       │
 * │     → Открыть файл на SD                                       │
 * │                                                                 │
 * │  2. UPLOAD_FILE_WRITE (много раз)                              │
 * │     upload.buf[] = данные (до 1460 байт)                       │
 * │     upload.currentSize = размер текущего чанка                 │
 * │     → Записать на SD                                           │
 * │                                                                 │
 * │  3. UPLOAD_FILE_END                                            │
 * │     upload.totalSize = общий размер                            │
 * │     → Закрыть файл, показать успех                             │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * 
 * ПРИМЕР UPLOAD HANDLER:
 * 
 * File uploadFile;
 * 
 * void handleUploadPost() {
 *   HTTPUpload& upload = server.upload();
 *   
 *   if (upload.status == UPLOAD_FILE_START) {
 *     String filename = upload.filename;
 *     uploadFile = SD.open("/" + filename, FILE_WRITE);
 *     
 *   } else if (upload.status == UPLOAD_FILE_WRITE) {
 *     // Запись чанка на SD
 *     if (uploadFile) {
 *       uploadFile.write(upload.buf, upload.currentSize);
 *     }
 *     
 *   } else if (upload.status == UPLOAD_FILE_END) {
 *     if (uploadFile) {
 *       uploadFile.close();
 *     }
 *   }
 * }
 * 
 */

/*
 * 4.1. CHUNKED TRANSFER ENCODING
 * -------------------------------
 * 
 * HTTP/1.1 поддерживает передачу данных по частям без знания
 * общего размера заранее.
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │                 Chunked Transfer Example                        │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  HTTP/1.1 200 OK                                                │
 * │  Transfer-Encoding: chunked                                    │
 * │  Content-Type: text/html                                       │
 * │                                                                 │
 * │  1a\r\n           (26 байт в hex)                              │
 * │  <html><body>Hello\r\n                                         │
 * │  10\r\n           (16 байт)                                    │
 * │  World!</body></html>\r\n                                      │
 * │  0\r\n            (конец)                                      │
 * │  \r\n                                                           │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * В ESP8266WebServer это делается автоматически через streamFile():
 * 
 * File file = SD.open("/large_file.zip");
 * server.streamFile(file, "application/zip");
 * // Автоматически использует chunked encoding
 * 
 * 
 * ПОЧЕМУ ЭТО ВАЖНО ДЛЯ ESP8266:
 * 
 * 1. Не нужно знать размер файла заранее
 * 2. Не нужно загружать весь файл в RAM
 * 3. Передаём по мере чтения с SD
 * 4. Работает с файлами ЛЮБОГО размера
 * 
 * 
 * ДИАГРАММА ПОТОКА:
 * 
 * ┌───────────┐     read()      ┌───────────┐     send()     ┌───────────┐
 * │ SD-карта  │ ──────────────► │  ESP8266  │ ─────────────► │  Клиент   │
 * │           │   512-4096B     │  Buffer   │    1460B       │  Browser  │
 * │  Файл     │                 │  ~1.5KB   │    TCP packet  │           │
 * └───────────┘                 └───────────┘                └───────────┘
 *      ▲                             │
 *      │                             │
 *      └─────────────────────────────┘
 *         Цикл: читаем → отправляем → повторяем
 * 
 */


/*
 * 4.2. STREAMING FILE TRANSFER
 * -----------------------------
 * 
 * Ключевая оптимизация для работы с большими файлами.
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │              Буферизированная vs Потоковая                      │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  БУФЕРИЗИРОВАННАЯ (ПЛОХО для больших файлов):                  │
 * │                                                                 │
 * │  ┌─────────────────────────────────────────────────────┐       │
 * │  │  RAM                                                 │       │
 * │  │  ┌─────────────────────────────────────────────┐    │       │
 * │  │  │  Весь файл загружен в память                 │    │       │
 * │  │  │  10 MB файл = 10 MB RAM                      │    │       │
 * │  │  │  → OUT OF MEMORY!                            │    │       │
 * │  │  └─────────────────────────────────────────────┘    │       │
 * │  └─────────────────────────────────────────────────────┘       │
 * │                                                                 │
 * │  ПОТОКОВАЯ (ХОРОШО):                                           │
 * │                                                                 │
 * │  ┌─────────────────────────────────────────────────────┐       │
 * │  │  RAM                                                 │       │
 * │  │  ┌──────────────────────────────┐                   │       │
 * │  │  │  Маленький буфер ~1.5 KB     │                   │       │
 * │  │  │  Читаем → Отправляем → Читаем│                   │       │
 * │  │  │  → Работает с файлами любого размера            │       │
 * │  │  └──────────────────────────────┘                   │       │
 * │  └─────────────────────────────────────────────────────┘       │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * 
 * РЕАЛИЗАЦИЯ:
 * 
 * void handleDownload() {
 *   File file = SD.open(path, FILE_READ);
 *   
 *   // Вариант 1: Автоматический streaming
 *   server.streamFile(file, getMimeType(path));
 *   
 *   // Вариант 2: Ручной streaming (больше контроля)
 *   server.setContentLength(file.size());
 *   server.sendHeader("Content-Disposition", 
 *                     "attachment; filename=" + filename);
 *   server.send(200, getMimeType(path), "");
 *   
 *   uint8_t buffer[1460];
 *   while (file.available()) {
 *     size_t len = file.read(buffer, sizeof(buffer));
 *     server.client().write(buffer, len);
 *     yield(); // Важно: даём WiFi стеку работать
 *   }
 *   
 *   file.close();
 * }
 * 
 * 
 * ВАЖНО: yield() В ЦИКЛАХ!
 * 
 * WiFi стек ESP8266 требует регулярного внимания.
 * Если код занят в цикле более 50 мс без yield(),
 * WiFi соединение разорвётся.
 * 
 * while (file.available()) {
 *   // ... чтение/запись ...
 *   yield();  // Даёт WiFi стеку обработать события
 * }
 * 
 */


/*
 * 4.3. MIME-ТИПЫ
 * ---------------
 * 
 * MIME (Multipurpose Internet Mail Extensions) - стандарт для
 * указания типа содержимого.
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │                    Common MIME Types                            │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  Текстовые:                                                    │
 * │  ├── text/html           .html, .htm                           │
 * │  ├── text/css            .css                                  │
 * │  ├── text/javascript     .js                                   │
 * │  ├── text/plain          .txt, .log, .md                       │
 * │  ├── application/json    .json                                 │
 * │  └── application/xml     .xml                                  │
 * │                                                                 │
 * │  Изображения:                                                  │
 * │  ├── image/jpeg          .jpg, .jpeg                           │
 * │  ├── image/png           .png                                  │
 * │  ├── image/gif           .gif                                  │
 * │  ├── image/svg+xml       .svg                                  │
 * │  └── image/webp          .webp                                 │
 * │                                                                 │
 * │  Аудио/Видео:                                                  │
 * │  ├── audio/mpeg          .mp3                                  │
 * │  ├── audio/wav           .wav                                  │
 * │  ├── video/mp4           .mp4                                  │
 * │  └── video/webm          .webm                                 │
 * │                                                                 │
 * │  Документы:                                                    │
 * │  ├── application/pdf     .pdf                                  │
 * │  └── application/zip     .zip                                  │
 * │                                                                 │
 * │  Binary:                                                       │
 * │  └── application/octet-stream   (по умолчанию)                │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * 
 * ФУНКЦИЯ getMimeType():
 * 
 * String getMimeType(String path) {
 *   path.toLowerCase();
 *   
 *   if (path.endsWith(".html")) return F("text/html");
 *   if (path.endsWith(".css"))  return F("text/css");
 *   if (path.endsWith(".jpg"))  return F("image/jpeg");
 *   // ... и так далее
 *   
 *   return F("application/octet-stream");  // Binary по умолчанию
 * }
 * 
 * 
 * ПОЧЕМУ ЭТО ВАЖНО:
 * 
 * 1. Браузер знает как отображать файл
 *    text/html → рендерит страницу
 *    image/jpeg → показывает картинку
 *    application/octet-stream → предлагает скачать
 * 
 * 2. Правильные заголовки = правильное поведение
 *    Content-Type: text/plain → показывать как текст
 *    Content-Type: application/json → можно парсить как JSON
 * 
 */

/*
 * 5.1. АВТОРИЗАЦИЯ
 * -----------------
 * 
 * Простая HTTP Basic Auth + Session Cookie.
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │                    Auth Flow                                    │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  1. Неавторизованный запрос                                    │
 * │     GET / HTTP/1.1                                             │
 * │     Cookie: (none)                                             │
 * │                                                                 │
 * │  2. Сервер редиректит на /login                                │
 * │     HTTP/1.1 302 Found                                         │
 * │     Location: /login                                           │
 * │                                                                 │
 * │  3. Пользователь вводит логин/пароль                           │
 * │     POST /login HTTP/1.1                                       │
 * │     user=admin&pass=admin                                      │
 * │                                                                 │
 * │  4. Сервер проверяет и выдаёт cookie                           │
 * │     HTTP/1.1 302 Found                                         │
 * │     Set-Cookie: session=a1b2c3d4e5f6...                        │
 * │     Location: /                                                │
 * │                                                                 │
 * │  5. Последующие запросы с cookie                               │
 * │     GET / HTTP/1.1                                             │
 * │     Cookie: session=a1b2c3d4e5f6...                            │
 * │                                                                 │
 * │  6. Сервер проверяет cookie и пускает                          │
 * │     HTTP/1.1 200 OK                                            │
 * │     ... контент ...                                            │
 * │                                                                 │
 * └─────────────────────────────────────────────────────────────────┘
 * 
 * 
 * РЕАЛИЗАЦИЯ:
 * 
 * // Проверка авторизации
 * bool checkAuth() {
 *   if (server.hasHeader("Cookie")) {
 *     String cookie = server.header("Cookie");
 *     if (cookie.indexOf("session=" + sessionToken) != -1) {
 *       return true;
 *     }
 *   }
 *   return false;
 * }
 * 
 * // Генерация токена
 * String generateToken() {
 *   String token = "";
 *   for (int i = 0; i < 16; i++) {
 *     token += String(random(16), HEX);
 *   }
 *   return token;
 * }
 * 
 * // Выход
 * void handleLogout() {
 *   sessionToken = "";
 *   server.sendHeader("Set-Cookie", "session=; Path=/; Max-Age=0");
 *   server.sendHeader("Location", "/login");
 *   server.send(302, "text/plain", "");
 * }
 * 
 * 
 * ОГРАНИЧЕНИЯ:
 * 
 * 1. Токен хранится в RAM - теряется при перезагрузке
 * 2. Нет HTTPS - пароль передаётся открытым текстом
 * 3. Один токен на всех - только один активный пользователь
 * 
 * Для продакшена нужно:
 * - HTTPS (требует больше памяти)
 * - Хранение токенов в EEPROM
 * - Срок действия токена
 * 
 */


/*
 * 5.2. ЗАЩИТА ФАЙЛОВОЙ СИСТЕМЫ
 * -----------------------------
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │                  Security Measures                              │
 * ├─────────────────────────────────────────────────────────────────┤
 * │                                                                 │
 * │  1. PATH TRAVERSAL ЗАЩИТА                                      │
 * │                                                                 │
 * │     Атака: GET /download?path=../../etc/passwd                 │
 * │                                                                 │
 * │     Защита:                                                     │
 * │     String path = server.arg("path");                          │
 * │     if (path.indexOf("..") >= 0) {                             │
 * │       path = "/";  // Безопасный путь по умолчанию             │
 * │     }                                                           │
 * │                                                                 │
 * │                                                                 │
 * │  2. ЗАПРЕТ УДАЛЕНИЯ КОРНЯ                                      │
 * │                                                                 │
 * │     if (path == "/") {                                          │
 * │       server.send(400, "text/plain", "Cannot delete root");    │
 * │       return;                                                   │
 * │     }                                                           │
 * │                                                                 │
 * │                                                                 │
 * │  3. ПРОВЕРКА СУЩЕСТВОВАНИЯ ПЕРЕД ДЕЙСТВИЕМ                     │
 * │                                                                 │
 * │     if (!SD.exists(path)) {                                     │
 * │       server.send(404, "text/plain", "Not found");             │
 * │       return;                                                   │
 * │     }                                                           │
 * │                                                                 │
 * │                                                                 │
 * │  4. УДАЛЕНИЕ ТОЛЬКО ПУСТЫХ ПАПОК                                │
 * │                                                                 │
 * │     SD.rmdir(path);  // Удаляет только пустые папки            │

/*
 * ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ ESP8266:
 * 
 * 1. ПАМЯТЬ
 *    - Максимум ~40-50 KB свободной RAM
 *    - HTML страницы должны храниться во Flash
 *    - Нельзя загрузить весь файл в память
 * 
 * 2. WIFI
 *    - Только 2.4 GHz
 *    - Максимум 4-8 одновременных клиентов
 *    - Нет 5 GHz
 * 
 * 3. ФАЙЛОВАЯ СИСТЕМА
 *    - Только FAT32
 *    - Длинные имена файлов могут обрезаться
 *    - Нет прав доступа (Linux-style)
 * 
 * 4. ПРОИЗВОДИТЕЛЬНОСТЬ
 *    - Скорость передачи: 200-400 KB/s
 *    - Однопоточная обработка запросов
 *    - Нет параллельных соединений
 * 
 * 5. БЕЗОПАСНОСТЬ
 *    - Нет аппаратного HTTPS
 *    - Пароли передаются открытым текстом
 *    - Нет шифрования данных
 * 
 * 
 * РЕКОМЕНДАЦИИ ПО ПРЕОДОЛЕНИЮ:
 * 
 * ┌─────────────────────────────────────────────────────────────────┐
 * │ Проблема           │ Решение                                   │
 * ├─────────────────────────────────────────────────────────────────┤
 * │ Мало памяти        │ PROGMEM, streaming, F() макрос            │
 * │ Медленная передача │ Chunked encoding, параллельная обработка  │
 * │ Нет HTTPS          │ Использовать VPN или туннель              │
 * │ Мало клиентов      │ ESP32 (до 10 клиентов)                    │
 * │ FAT32 только       │ Использовать LittleFS для встроенной Flash│
 * └─────────────────────────────────────────────────────────────────┘
 */
